Accounts Module Help

About the Accounts Module

The Accounts module in AcademEA provides comprehensive financial management tools for educational institutions. It allows you to manage student fees, track payments, generate financial reports, and monitor income and expenditure.

This help documentation covers the following specific features of the Accounts module:

  • Termly Bill: Generate and manage term bills for students
  • Fee Payment Status: Track payment status of students
  • Receive Fee / School Fees: Process school fee payments from students
  • Receive Feeding Fee (Daily/Weekly): Process daily or weekly feeding fee payments
  • Receive Feeding Fee (Termly): Process termly feeding fee payments
  • Daily Cash / All Income: View all income transactions
  • Daily Cash / School Fees: View school fee transactions specifically

How to Navigate to Accounts Module

Log in to your AcademEA account with appropriate privileges.

From the main menu, click on Accounts.

You will see various options including Term Bill, Fee Payment Status, Receive Fee, and Daily Cash.

Select the specific feature you want to use.

Accounts Module Navigation

Termly Bill

The Termly Bill feature allows you to generate and manage bills for students for a specific term. You can view, print, and manage these bills.

How to Generate Termly Bills

Navigate to Accounts → Term Bill.

Select the academic year and term for which you want to generate bills.

Select the class or specific students you want to generate bills for.

Click "Generate Bills" to create the termly bills.

Review the generated bills and make any necessary adjustments.

Print or save the bills as needed.

Termly Bill Interface

Fee Payment Status

The Fee Payment Status feature allows you to view and monitor the payment status of students. You can filter by class, student status, and other criteria.

How to Check Fee Payment Status

Navigate to Accounts → Fee Payment Status.

Use the search function to find a specific student or browse the list.

Filter by class using the "Filter by Class" dropdown.

Check "Show old students" to include former students in the results.

View the payment status and balance for each student.

Fee Payment Status Interface

Receive Fee / School Fees

The Receive Fee feature allows you to process school fee payments from students. You can accept payments through various methods including cash, cheque, mobile money, and bank transfers.

How to Receive School Fee Payments

Navigate to Accounts → Receive Fee → School Fees.

Select a student from the list or use the search function to find a specific student.

Enter the payment amount in the "Amount" field.

Select the payment method (Cash, Cheque, Bank Draft, Pay in Slip, or Momo).

Add a description for the payment receipt.

Click "Preview Payment" to review the details before finalizing.

Confirm the payment to process it.

Receive School Fees Interface

Payment Methods

The system supports multiple payment methods:

  • Cash: Direct cash payments
  • Cheque: Payment by cheque (requires cheque number)
  • Bank Draft: Bank draft payments
  • Pay in Slip: Bank deposit slips
  • Momo: Mobile money payments (requires mobile number)

Receive Feeding Fee (Daily/Weekly)

The Receive Feeding Fee (Daily/Weekly) feature allows you to process payments for student feeding programs on a daily or weekly basis.

How to Receive Daily/Weekly Feeding Fees

Navigate to Accounts → Receive Fee → Feeding Fee → Daily/Weekly.

Select a student from the list or search for a specific student.

Select the academic year and term.

Choose the specific day or week for which you're collecting fees.

Enter the amount for the selected period.

Select the payment method and date.

Review and confirm the payment.

Receive Feeding Fee (Daily/Weekly) Interface

Receive Feeding Fee (Termly)

The Receive Feeding Fee (Termly) feature allows you to process payments for student feeding programs for an entire term.

How to Receive Termly Feeding Fees

Navigate to Accounts → Receive Fee → Feeding Fee → Termly.

Select a student from the list or search for a specific student.

Select the academic year and term.

The system will display the total feeding fee for the term.

Enter the payment amount (full or partial payment).

Select the payment method and date.

Review and confirm the payment.

Receive Feeding Fee (Termly) Interface

Daily Cash / All Income

The Daily Cash / All Income feature provides a comprehensive view of all income transactions processed on a specific day or date range.

How to View All Income Reports

Navigate to Accounts → Daily Cash → All Income.

Select a start date and end date for the report.

Use the search function to find specific transactions if needed.

View the list of all income transactions with details including date, receipt number, student information, and amount.

Export the report if needed for record-keeping or accounting purposes.

All Income Interface

Daily Cash / School Fees

The Daily Cash / School Fees feature provides a detailed view of all school fee transactions processed on a specific day or date range.

How to View School Fee Transactions

Navigate to Accounts → Daily Cash → School Fees.

Select a start date and end date for the report.

Use the search function to find specific transactions if needed.

View the list of school fee transactions with details including date, receipt number, student information, and amount.

Export the report if needed for record-keeping or accounting purposes.

Best Practices for Accounts Management

Regular Reconciliation: Regularly reconcile your accounts to ensure accuracy.

Backup Records: Regularly backup financial records for security and audit purposes.

User Permissions: Assign appropriate permissions to staff members based on their roles.

Regular Reports: Generate and review financial reports regularly to monitor the institution's financial health.

Important Notes

  • Only users with appropriate permissions can access financial features.
  • All financial transactions are logged for audit purposes.
  • Always verify payment details before processing transactions.
  • Regularly update fee structures to reflect current rates.

Troubleshooting Common Issues

Issue Possible Cause Solution
Cannot process payment Insufficient permissions or system error Check your user permissions or contact system administrator
Incorrect balance shown Pending transactions not processed Refresh the page or check for pending transactions
Student not appearing in list Student may be inactive or filtered out Check filters and student status
Report data not accurate Date range may be incorrect Verify the date range selected for the report
