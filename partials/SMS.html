SMS Module

Overview

The SMS module allows you to communicate directly with parents and guardians through text messages. This feature can be accessed from the side menu of the AcademEA system.

Figure 1: SMS menu location

Send SMS

This feature allows you to send SMS messages to selected parents or guardians.

Figure 2: Send SMS interface
  1. Navigate to the "Send SMS" section from the SMS menu.
  2. Select the recipients by choosing specific classes or individual students.
    3. Figure 3: Selecting classes for SMS
  3. Enter any additional numbers separated by commas to receive the message (e.g., 0200000001,0244000001).
  4. Compose your message in the provided text area.
    5. Figure 4: Message composition area
  5. Optionally, check 'Add All Parents From Selected Class' to include all parents automatically.
  6. Review your message and ensure all details are correct.
  7. Click the "Send" button to dispatch the SMS to the selected recipients.

Tip

Keep messages concise to stay within the single SMS character limit (160 characters) and avoid additional charges for multiple messages.

SMS Arrears to Parents

This feature allows you to notify parents of students who owe school fees. The system automatically lists students with arrears along with their class, amount owed, and parent phone number.

Figure 5: SMS Arrears interface
  1. Select the Class/Form from the dropdown menu.
    2. Figure 6: Selecting a class for arrears notification
  2. Review the table of students with arrears, including ID, name, class, amount, and phone number.
    3. Figure 7: List of students with fee arrears
  3. Click 'Send SMS' to notify the parents directly from the system.

Important

Ensure that parent phone numbers are correctly recorded in the system before sending arrears notifications.

SMS Configuration (API Setup)

For SMS to work, the system must be connected to an SMS gateway service. This is done in the Configuration section.

Understanding SMS Configuration

API ID and API Password: These are provided by your SMS service provider (e.g., Vodafone, MTN, BulkSMS.com). They act as a username and password to send messages through their gateway.

Sender ID: This is the name that will appear on the recipient's phone as the sender (e.g., MountEyeSch). This often needs to be registered and approved with the provider.

Figure 10: SMS configuration interface

To Add/Edit Configuration:

  1. Go to SMS > Configuration.
  2. Enter the API ID, API Password, and approved Sender ID provided by your SMS company.
    3. Figure 11: SMS configuration form
  3. Click "Submit" to save the details or "Reset" to clear the form.
    4. Figure 12: Configuration submission buttons

Important

If you haven't chosen an SMS provider yet, you will need to sign up for one to get these credentials. Without correct configuration, the SMS features will not function.

SMS Templates

This feature helps you create and manage pre-written messages for quick, consistent communication.

Figure 13: SMS templates interface

To Add a New Template:

  1. Click "Add" at the top right or look for the creation form.
  2. Template Name: Enter a unique, descriptive name (e.g., "Exam Schedule Alert").
  3. Template: Type the full body of your standard message.
  4. Active: Set the dropdown to "Yes" to activate the template and make it selectable when sending messages.
  5. Click "Submit" to save.
    6. Figure 14: Saving an SMS template

Usage Tip

When in the "Send SMS" section, you will typically have an option to load a saved template to pre-fill the message box, saving time and ensuring message consistency.
