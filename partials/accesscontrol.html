Access Control Help

About Access Control

The Access Control module allows you to manage user permissions and security within the AcademEA system. This feature ensures that staff members have appropriate access levels based on their roles and responsibilities.

The Access Control module consists of three main components:

  • User Groups: Define roles with specific permission sets
  • User Access: Manage permissions for different user groups
  • Users: Create and manage individual user accounts

How to Navigate to Access Control

Log in to your AcademEA account with administrative privileges.

From the main menu, navigate to System Setup.

Within System Setup, select Access Control.

You will be directed to the Access Control dashboard with options for User Groups, User Access, and Users.

Navigation Path Preview

User Groups

User Groups allow you to define roles with specific permission sets. Common groups include Administrator, Accountant, Instructor, and Cashier.

How to Create a User Group

Navigate to Access Control → User Groups.

Click the Add button to create a new group.

Enter a name for the new group in the "Group Name" field.

Click Submit to save the new group or Reset to clear the form.

User Access

The User Access section allows you to define permissions for each user group, controlling what features and data they can access.

How to Configure User Access Permissions

Navigate to Access Control → User Access.

Select the user group you want to configure from the dropdown menu.

Check or uncheck the permissions you want to grant or revoke for that group.

Click Update to save your changes.

Users

The Users section allows you to create and manage individual user accounts for staff members.

How to Add a New User

Navigate to Access Control → Users → Add User.

Select a staff member from the dropdown or enter their details manually.

Enter a User ID and Email address for the new user.

Select the appropriate Security Group from the dropdown.

Set a strong password that meets the system requirements.

Confirm the password and set the account status to Active.

Click Submit to create the user or Cancel to discard.

Add User Interface

How to View and Manage Users

Navigate to Access Control → Users → Display Users.

Use the search function to find specific users or browse the list.

Click the edit icon to modify user details or permissions.

Click the delete icon to remove a user (requires confirmation).

Password Requirements

When creating user accounts, passwords must meet the following security requirements:

  • At least 8 characters
  • 1 lowercase and 1 uppercase letter
  • 1 number (0-9)
  • 1 special character (!@#$%^&*)

Best Practices for Access Control

Principle of Least Privilege: Grant users only the permissions they need to perform their job functions.

Regular Audits: Periodically review user accounts and permissions to ensure they're still appropriate.

Strong Passwords: Enforce password complexity requirements and regular password changes.

Deactivate Inactive Accounts: Disable accounts for staff members who no longer need system access.

Important Notes

  • Only users with administrative privileges can access the Access Control module.
  • Changes to user permissions take effect immediately after saving.
  • It's recommended to test new permission sets before applying them to multiple users.
  • Keep backup administrator accounts in case of accidental lockout.

Troubleshooting Common Issues

Issue Possible Cause Solution
User cannot access certain features Insufficient permissions assigned to their user group Check and update the user group permissions in User Access
Password not accepted Password doesn't meet complexity requirements Ensure password meets all requirements (length, character types)
User account not appearing in list Account may be inactive or filtered out Check active status and search filters
Cannot modify certain permissions Your account may lack necessary privileges Contact a system administrator with higher privileges
